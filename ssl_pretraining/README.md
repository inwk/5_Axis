# SSL Pretraining

This folder contains self-supervised learning utilities that are separate from
CAM transition collection/training.

## Files

- `collect_prt_state_encoder_dataset.py`
  - NX/Open script.
  - Opens `.prt` files and writes lightweight StateEncoder SSL parquet files.
  - Default output mode is one parquet per `.prt` so rows are not accumulated in memory.
  - Set `WRITE_ONE_PARQUET_PER_PART = False` only if a single bundled parquet is explicitly needed.
  - Does not create CAM operations, IPW bodies, toolpaths, octree labels, or SDF transition labels.
  - `state_points[..., 6]` is zero-filled because prt-only data has no current residual/SDF.

- `train_state_encoder_ssl.py`
  - PyTorch training script for StateEncoder SSL.
  - Reads the parquet generated by the extractor above or the full CAM parquet dataset.
  - Saves `encoder_state_dict` for later transfer into transition training.
  - Uses a dedicated face-type mask token by default (`MASK_FACE_TYPE_ID = -1`
    means `face_type_vocab_size - 1`) so real class `0` is not reused as mask.

- `analyze_face_type_distribution.py`
  - Reads only `face_type_512`, `node_mask`, and optional `part_name`.
  - Reports valid-node face-type counts, ratios, part coverage, and class-balance weights.

- `state_encoder_dataset.py`
  - Minimal parquet dataset loader for StateEncoder SSL columns.

- `state_encoder_model.py`
  - StateEncoder SSL heads and loss functions.

## Recommended prt-only workflow

1. Set `PRT_DIR` and `OUTPUT_DIR` in `collect_prt_state_encoder_dataset.py`.
2. Run it inside NX/Open Python.
3. Set `PARQUET_DIR` or `EXPLICIT_PARQUET_PATHS` in `train_state_encoder_ssl.py`.
4. Keep `SDF_LOSS_WEIGHT = 0.0` for prt-only data.
5. Train and use the saved `encoder_state_dict` for downstream transition fine-tuning.

Run `analyze_face_type_distribution.py` before training if you need to inspect
face-type class imbalance.
